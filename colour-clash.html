Colour Clash

Overview

As part of a group project for my "Prototyping Physical Interaction" course, we developed a 2-player color reaction game. The game is designed to be competitive yet fun and simple, targeting players of all ages. The goal was to create an engaging game that also helps improve players' reaction times over repeated play sessions.

My Role:

In this project, I was primarily responsible for documentation and user research. This involved gathering and analyzing user feedback to inform the iterative development process and ensure a user-centered design.

User Research and Insights:

I conducted comprehensive user research through playtesting sessions and documented the following key insights:

  • Color Distinction Issues: Users found it challenging to differentiate between cyan and blue on the keypads.
  • Terminology Confusion: Some users were unfamiliar with the term "magenta" and suggested renaming it to "purple."
  • Ergonomic Preferences: Feedback indicated a preference for holding the keypads separately rather than having them fixed on the board.
  • Screen Size Concerns: Several users felt the LCD display was too small.
  • Game Balance: The existing point deduction system created a negative feedback loop for losing players. Users suggested introducing a streak system to balance the gameplay.
  • Sound Feedback: The sounds for correct and incorrect inputs were indistinguishable and perceived as distracting.

Implemented Changes

Based on the user feedback, our team made several iterations to improve the game:

  • Display Pattern Standardization: We standardized the Neotrellis board display to ensure a fair distribution of colors for both players.
  • Keypad Design: Redesigned keypads to be held separately in each hand, enhancing user comfort and versatility.
  • Color Adjustments: Changed cyan to orange for better visibility and renamed "magenta" to "purple."
  • Combo System: Implemented a combo streak system that rewards consecutive correct answers, making the game more balanced and competitive.
  • Sound Feedback: Modified buzzer sounds to clearly differentiate between correct and incorrect inputs.

Final Outcome and Further Improvements

The final prototype incorporated these changes, resulting in a more engaging and user-friendly game. Key features included:

  • Neotrellis Boards: Attached to 3D casings with hidden wires, power supply, and buzzer.
  • LCD Screen: Centrally placed to display scores and colors.
  • Ergonomic Keypads: Players can hold the color boards, enabling two-handed play.
  • Distinctive Colors: Seven easily distinguishable colors (red, green, orange, yellow, purple, blue, white) were used.
  • Enhanced Sound Feedback: Clear audio signals for correct and incorrect inputs.

Despite these improvements, further enhancements could be made based on additional user feedback, such as increasing the size of the LCD screen or implementing dual screens for each player.
